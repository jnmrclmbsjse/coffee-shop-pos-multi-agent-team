Design notes

Exercise prototype states

Interaction contract

  • Remembered staff select a display-name tile, then enter exactly four digits. Sign in stays disabled until all four cells are filled.
  • Backspace removes one digit. Clear removes all digits. Their availability follows the current PIN length.
  • Prototype PIN 1234 succeeds. Other complete PINs use the generic failure. Username admin is also rejected generically.
  • A completed Username and Password sign-in remembers the staff member on this browser/device. Use username fail to exercise failure.
  • Submitting locks every credential control to prevent double-submit.

Responsive and touch behavior

  • At 1024×768, identity and keypad sit side by side. At 768×1024, identity condenses above the keypad.
  • Every digit, Clear, Backspace, staff tile, text action, and submit control is at least 44×44 CSS px. Fields are exactly 48px tall.
  • At both acceptance tablet sizes, the full PIN keypad and submit button remain visible without page scrolling. Mobile uses a single-column staff list and stacked PIN layout.

Accessibility

  • Semantic sections and form labels, visible focus rings, high-contrast tokens, icon-only aria labels, and polite aria-live messaging.
  • The masked PIN exposes a text equivalent such as “2 of 4 PIN digits entered.”
  • Number keys also work from the physical keyboard. Backspace, Delete, and Enter mirror their visible controls.
  • Reduced-motion preferences shorten all feedback to an effectively instant state change.

Security and QA copy

Failures never distinguish wrong, unassigned, deactivated, administrator, or nonexistent credentials.
